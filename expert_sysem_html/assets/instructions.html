Installation

Offline

From our downloads page you can navigate to the link to download the Expert System Builder for offline use. The download will be a zip archive. Unzip the archive. The zip archive will contain a README, the HTML for the expert system, and dependecy folders which allow the expert system to run. You can run the expert system by opening the HTML in a supported internet browser.

Getting Started

From the top ribbon select "App" to get to the Expert User Interface. You will be presented with various ways to start using an expert system. "New System" let’s you just jump right in and create an Expert System from scratch. "Continue” sends you back to your saved progress if you were already working on a system. On the "Upload System" area, you can edit a pre-fabricated Expert System that was built using this application, or use it to give a prediction. The upload only receives a .json type of file that has a previously created Expert System. Once the .json is uploaded, you can use the two buttons in the area. The "Start Expert Interface” button populates the Expert side with rules, attributes and actions that can be seen and edited on the summary tab. The "Start User Interface” button takes you to the page where the Expert System makes a prediction. You can also choose to load a template of a pre-created Expert System to start from there using the Pre Built System area. If you choose "Sample Expert Interface" it will take you to an already populated expert side where you can see the rules, attributes, and actions in the summary Area. If you choose "Sample User Interface" it will take you through an example on how it works for the end-user side.

Expert User Instructions

Navigating the Expert User Interface

The interface has different tabs that can be used according to the user's preference

Info

When you open the expert user interface you will start at the "Info" tab. From here you may name your Expert System, add a description of the system, and include author names for the system.

Attributes

The "Attributes" tab allows you to create attributes. Attributes are like the variables of a system. Every attribute has a name which you can use to identify which attribute is being worked on. The question field is the question that will be prompted to the user using the expert system when trying to obtain a value for an attribute. Values are the possible values an attribute can have. Values have 4 different types, "multiple", "single", "numeric", and "boolean."

Multiple allows a user to select any number of values for an attribute at a time. For example, you might have the attribute "clothes" and the question "What clothes are you wearing?" A user could select the values "t-shirt', "pants", "socks" from a list of values if they are fully clothed. If a user is not wearing clothes they do not have to select any values.

Single allows a user to only select a single value from a list of values. For example, you might have the attribute "weather" with possible values "sunny," "rainy," and "snowy" A user can only select one value at a time because it does not make sense to have both "sunny" and "rainy" at the same time.

Numeric means an attribute holds a numeric value. Numeric values also allow you to specify a range of acceptable values to enter and whether valid values are integers or decimal values. Numeric ranges can also be exclusive (meaning you cannot select the end values of the range) or inclusive (meaning you can select the end value of a range). An example of a numeric value would be the attribute "age" which only allows integers in a range from 0 to 120. A user using the expert system will be able to key in an age which is within the range and is an integer.

Boolean means an attribute holds either the value "yes" or "no." You cannot specify any other values for a boolean type. For example, you might have a boolean attribute that asks "Is your computer turned on?." The user will see the question and be able to select "yes" or "no."

Rules

A rule has a name, if antecedent, and then consequent. A rule also allows the display of a message or attribute which will be run in the expert system if a rule is determined to be true. These display "commands" are explained in the actions section below. The name field allows the rule to be easily identified so an expert knows which rule they are working on.

If antecedent are the conditions of a rule which imply a consequent. For example you might have the rule, IF weather = raining AND loaction = outside Then condition = wet. The antecedent in this case are weather = raining AND location = outside. In this example, weather and location are the names of attributes, and raining and outside are the respective values that the attribute must have for the rule to be true. The consequent in this example is condition = wet where condition is an attribute and wet is a value being assigned to condition.

An antecedent can have each attribute value pair seperated by AND or OR. AND and OR function as the boolean orperators normally would with OR taking precedence over AND. This mean A AND B OR C is interpreted as A AND (B OR C). A consequent can have any number of attribute value assignment pairs seperated by AND. The consequent will only assign attributes value if the antecednet is true and the attribute is not already assigned a value.

Actions

Actions are the way a user is guided through an expert system. Actions are commands which are iterated through in order and presented to the user in the expert system. There are three possible actions, "Display Message", "Display Attribute", and "Obtain Attribute."

Display Message simply displays whatever text included to the user. For example you could have a display message command which says "Starting the expert system" at the beginning of the system. Messages are helpful to an expert to provide explanations for the flow of the expert system to the user accessing the system.

Display Attribute will display the value(s) of an attribute which have been assigned and display them to the user of the expert system. Display Attribute should only be used on an attribute that already has an assigned value and will otherwise report an error.

Obtain Attribute will attempt to obtain the value of an attribute without directly querying the user what the attribute is. For example, let's pretend there is an expert system which diagnosis issues with cars. If you are trying to figure out what is wrong with a car, you can't ask the user "What is wrong with the car?" because the user does not know! That is why they are using the expert system in the first place. When an Obtain Attribute command is used on an attribute, it will search through all the rules to find which rules have that attribute in their consequent. For each rule that the engine goes through, it will determine if the antecedent is true. Whenever the engine comes upon an unknown attribute in the antecedent, it will query the user for an attribute and continue once that value has been obtained.

Summary

The summary tab allows a user to view all the attributes, rules, and actions currently entered into the system. From this tab you can select an attribute, rule, or action to modify or delete.

Troubleshooting

If you have an issue with using the expert system you can read the instructions page to learn more about how the system works. If you ever make a mistake creating a rule, attribute, or action, you can go to the summary tab and delete or modify your mistake. If you think there is an issue with the Expert Builder System you can contact Group 159 to report issues with the system.

Saving Progress

You will be presented with multiple options for saving your expert system. You can save a JSON file which can be reuploaded into the system to restore your state. If your browser has the ability to use local storage the system may be saved there and you can resume working on the website at a later time without worrying about your progress being lost.

End User Instructions

Using an Expert System

If you are an end user of the expert system, you have either recieved an expert system archive from an expert or have been directed to a website to use an expert system. Either way, you should run a html page with opens the expert system in a browser. Follow the on-screen instructions and you should expct the Expert System to prompt you with questions one at a time which you will answer. Questions might be yes or no questions, multiple choice, or entering anumerical value. When you are satisified with your entry, you may press the submit button to continue your progress through the expert system. The reset button will clear your responses for the current question if you wish to reset your entry before submitting. The expert system terminates when it stops asking questions and displaying on screen instructions

If you are using an expert system which seems to terminate abruptly or provide inaccurate information, you should contact the expert who created the system. It is likely that an expert may not have tested every case and there is an error with their system that they missed.
